## Admin Roles and Access Control

### Who is the admin?

The developer holds **Super Admin** access. The client gets a **Client Admin** role.

### Role Breakdown

| Access Level | Developer | Client |
|---|---|---|
| **Source code** | Full access | No access |
| **Server/hosting** | Full access | No access |
| **Database** | Full access | No access |
| **Domain/DNS** | Full access | View only (or shared) |
| **Content editing** | Full access | Full access |
| **Media uploads** | Full access | Full access |
| **User management** | Full access | Limited (own staff only) |
| **Plugin/theme updates** | Full access | No access |
| **Analytics** | Full access | View only |
| **Payment gateway settings** | Full access | View only |
| **Backups** | Full access | No access |

### Why this matters

- Developer maintains control over the codebase and infrastructure
- Client can update their own text, images, and basic content
- Prevents accidental changes that could break the site
- If the client wants full ownership, everything can be transferred at project handoff (handover fee applies)

### Handover Options

|---|---|---|
| Partial handover (all packages) | Content admin access only | Included in project |
